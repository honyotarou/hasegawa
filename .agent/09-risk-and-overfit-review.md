# 09 Risk And Overfit Review

Step 4 と Step 5 で使用する。

## 1. 過剰適合チェック

- [ ] 特定入力にだけ通るハードコードがない
- [ ] エラー文言だけを合わせる実装になっていない
- [ ] テストに現れない分岐で仕様逸脱がない
- [ ] `any` や無条件成功で逃げていない
- [ ] 失敗系が本当に失敗することを確認した

## 2. 抜け道チェック

- [ ] `allFrames:false` 未指定のままになっていない
- [ ] `results[1]` 以降を誤使用していない
- [ ] secret を URL/ログへ出していない
- [ ] storage に患者データ本体を保持していない
- [ ] LockService 外で重複判定していない

## 3. 回帰リスク

- [ ] 正規化仕様変更が hash 判定に影響しない
- [ ] SW停止時に UI disabled が取り残されない
- [ ] GAS redirect 先ドメイン不足がない
- [ ] Trigger 遅延特性を運用側が理解できる記述がある

## 4. テスト資産判定

残す基準:

- [ ] 仕様契約を固定するテスト
- [ ] 重大障害を防ぐ回帰テスト
- [ ] 境界値/異常系の代表ケース

薄くする基準:

- [ ] 同値で重複し、片方で十分なテスト
- [ ] 実装詳細に過度依存し壊れやすいテスト

削除基準:

- [ ] 仕様価値がなく保守コストのみ高いテスト
- [ ] 契約を担保しない断片的なアサーション
