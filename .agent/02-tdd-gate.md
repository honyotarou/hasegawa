# 02 TDD Gate

本プロジェクトでは以下の順序を固定し、逆行を禁止する。

## Step 1: Red（テストのみ）

完了条件:

- [ ] 実装コードを追加/変更していない
- [ ] モック実装をプロダクションへ入れていない
- [ ] テストダブル（`vi.fn` / spy / stub）で外部依存を差し替えた
- [ ] テスト実行し、失敗を確認した
- [ ] 失敗ログを `.agent/08-run-log-template.md` 形式で記録した

必須証跡:

- 失敗テスト名
- 期待値/実測値
- エラー種別（バリデーション/通信/例外など）

## Step 2: テストのみコミット

完了条件:

- [ ] コミット対象がテストファイルのみ
- [ ] 実装ファイル差分が 0
- [ ] コミットメッセージに `test(red)` を含めた

## Step 3: Green（テスト不変）

完了条件:

- [ ] Step 1 で作成したテストを一切変更していない
- [ ] 実装を反復し全テストパス
- [ ] 反復回数（目安 2-4 周）を記録した
- [ ] 追加した分岐に対するテスト不足がない

## Step 4: Refactor（過剰適合監査）

完了条件:

- [ ] `.agent/09-risk-and-overfit-review.md` の全項目を確認
- [ ] 抜け道（ハードコード等）がないことを確認
- [ ] 失敗時の挙動が仕様通りであることを再確認

## Step 5: テスト資産の取捨選択

完了条件:

- [ ] 残すテストの理由を記録
- [ ] 薄くする/削除するテストの理由を記録
- [ ] 重要契約テストを削除していない

## Gate 失格条件

- Red ログなしで実装開始
- テスト変更で Green 到達
- 正常系偏重（失敗系 < 正常系）
- Given/When/Then 未記載
