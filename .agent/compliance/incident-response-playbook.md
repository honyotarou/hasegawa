# Incident Response Playbook

更新日: 2026-03-02

## 重大度定義
- Sev1: 個人情報漏えいの疑い、長時間停止
- Sev2: 送信不能/集計不能
- Sev3: 一部機能不具合（回避策あり）

## 初動（15分以内）
1. 事象記録（時刻/症状/影響範囲）
2. 影響拡大防止（送信停止、設定変更凍結）
3. 連絡（院内責任者、システム管理者）

## 解析
- batchId/doctorId単位で事象追跡
- GAS実行ログ、デプロイ履歴、直前コミット差分を確認

## 復旧
- 暫定対処（ロールバック/設定戻し）
- 恒久対処（修正、テスト、再デプロイ）
- 再発防止策をIssue化

## 事後報告
- 発生原因
- 影響件数
- 復旧時刻
- 再発防止策と期限
